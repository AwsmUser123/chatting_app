Idea behind the project:
Two parts: server part and client part.
The client types the name of the other communicating party, and if it is online,
the server will forward the new messages to them, after updating the communication file.
The server keeps track of the user accounts, and forwards the communication between
individual users.
As long as the server is up, it is possible to message the other party.

Client:
Login screen (enter username and password).
Select / create chat.
Type the text you want to be sent.
Quit if desired.

Server:
Create and maintain an account file.
Display server IP and port.
Listen for incoming information.
If received, update corresponding chat file.
Afterwards, send the new data to the listening applications.

Current step: automatic chat updates for online members. UI integration. List of online users.
